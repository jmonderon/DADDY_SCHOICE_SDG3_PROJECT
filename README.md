# DADDY_SCHOICE_SDG3_PROJECT

## Project Overview: HealthBite is a Food Calorie Tracker system specifically designed to serve individuals living with health complications, including but not limited to high blood pressure (hypertension), diabetes, obesity, and age-related dietary restrictions. Unlike conventional calorie-counting applications that focus primarily on fitness goals, HealthBite functions as a comprehensive health management system that aligns nutritional monitoring with medical guidance and safety. The primary objective of the system is to empower vulnerable user groups, such as the elderly, chronically ill patients, and individuals under strict dietary supervision, to make informed food choices daily. By combining meal logging, nutrient tracking, and condition-specific dietary alerts, HealthBite bridges the gap between general wellness tools and medically oriented nutrition management. At its core, the application allows users to log meals and automatically calculate calorie intake, macro-nutrient distribution. These tracked values are then compared against personalized health thresholds set according to the user's medical profile. When intake approaches or exceeds safe limits, for example, sodium intake for a hypertensive patient, the system generates real-time alerts and actionable recommendations. The system also places great emphasis on usability and accessibility, particularly for elderly users who may not be comfortable with complex digital interfaces. The design philosophy prioritizes large, readable text, straightforward navigation, and a minimal number of steps required to log a meal or view a health summary. This ensures that even users with limited technological familiarity can benefit from the platform without frustration or confusion. In summary, HealthBite is not merely a fitness utility but a patient-centered digital health tool intended to prevent dietary-related health complications, support clinical dietary recommendations, and ultimately contribute to the long-term well-being of its users.
Installation Guide Step 1: Install Visual Studio and Required Workloads Begin by downloading and installing Microsoft Visual Studio from the official Microsoft website (https://visualstudio.microsoft.com). During installation, the Visual Studio Installer will prompt you to select workloads. For HealthBite, you must enable the following workloads to ensure all necessary tools and libraries are available. •ASP.NET and web development — required for building and running the web application layer •.NET desktop development — required if the project includes any Windows Forms components •Data storage and processing — required for SQL Server integration and database tools After selecting the workloads, proceed with the installation. This may take several minutes. Once complete, launch Visual Studio to verify the installation was successful.

Step 2: Clone or Open the Project in Visual Studio Once Visual Studio is installed, you need to load the HealthBite project. If the project is hosted in a version control repository such as GitHub or Azure DevOps, you can clone it directly from within Visual Studio by navigating to File > Clone a Repository and entering the repository URL. Alternatively, if you have received the project as a compressed archive (.zip), extract it to a preferred local directory and open it by selecting File > Open > Project/Solution and navigating to the .sln solution file.

Step 3: Restore NuGet Packages HealthBite relies on several third-party libraries managed through NuGet, including database connectors, charting libraries for nutritional visualizations, and any UI component packages. Visual Studio typically restores these automatically when the project is first opened. However, if packages are missing or you see build errors related to missing references, you can manually trigger a restore by right-clicking on the Solution in Solution Explorer and selecting Restore NuGet Packages.

Step 4: Configure the Database Connection String The application communicates with a SQL Server or MySQL database to store user profiles, meal logs, food data, and health alerts. The database connection string must be configured before the application can run. Open the Web.config file (for ASP.NET Web Forms) or appsettings.json (for ASP.NET Core) located in the root of the project. Locate the connectionStrings section and update it with your database server details.

Step 5: Create and Migrate the Database With the connection string in place, you must initialize the HealthBite database schema. The project includes a SQL script file located in the Database folder of the solution. Open Microsoft SQL Server Management Studio (SSMS) or the Visual Studio SQL editor, connect to your database server, and execute the provided script to create all required tables, stored procedures, and initial seed data.

Step 6: Build the Project Before deploying or running the application, build the entire solution to ensure there are no compilation errors. In Visual Studio, go to the Build menu and select Build Solution, or use the keyboard shortcut Ctrl + Shift + B. The Output window at the bottom of the screen will display the build results. Resolve any errors listed before proceeding.

Step 7: Run Locally Using IIS Express To test the application on your local machine before web deployment, Visual Studio includes a built-in lightweight web server called IIS Express. Press F5 or click the green Run button at the top of Visual Studio to launch the application. Visual Studio will start IIS Express, compile and host the application, and automatically open your default web browser navigating to the local address, typically http://localhost:PORT where PORT is assigned automatically.

Step 8: Publish to a Web Server (IIS Deployment) Once local testing is complete, you can publish the application to a live web server running Internet Information Services (IIS). In Visual Studio, right-click on the project in Solution Explorer and select Publish. A wizard will guide you through the deployment configuration. Select Web Server (IIS) as the publish target and enter your server's address, site name, and credentials. Visual Studio will package the compiled application, including all DLLs, static assets, configuration files, and views, and transfer them to the IIS server via Web Deploy or File System copy. Once published, open IIS Manager on the server, locate the HealthBite site, and ensure the application pool is configured to use the correct .NET CLR version matching your project target — either .NET Framework 4.8 or .NET Core (No Managed Code for .NET 6/7).
